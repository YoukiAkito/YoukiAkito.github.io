这段时间，我对这个博客做了一轮比较完整的整理。它没有变成一个复杂的网站，仍然是一个安静写作、阅读和回顾的地方；只是一些原本会打断阅读的小细节，被逐步修顺了。

## 顶栏导航

最明显的变化在顶栏。原来的导航比较基础，后来改成了胶囊式导航，并把明暗模式切换放在导航左侧。这样在电脑上看时，右上角的操作区域更集中，也更像一个完整的控制区。

移动端则做了不同处理。明暗模式切换不再折叠进菜单，而是始终留在顶栏；菜单按钮点开后，页面导航会以悬浮面板的方式显示，不再把页面内容向下推。这个改动让手机上的操作更轻，也减少了布局跳动。

## 文章目录

单篇文章页右侧增加了目录。文章里有二级、三级标题时，目录会自动生成；点击目录项，就能直接跳到对应段落。长文章读到一半时，读者不需要反复滚动找位置。

全部文章页也增加了一个左侧目录。它列出当前页面里的文章标题，点击后会跳到对应文章卡片。这个功能比较适合文章逐渐变多以后使用，既能快速浏览，也能帮助定位。

## 页面宽度

增加目录后，文章页和全部文章页都变成了左右两栏布局。为了避免顶栏和正文左右边缘不一致，我统一了这些页面的整体宽度，包括顶栏容器、正文区域和目录区域。

窄屏下目录会自动隐藏，页面回到单栏阅读模式。这样电脑上有目录，手机上不挤占阅读空间。

## 明暗模式

暗色模式做了几处细节修正。导航、标签、目录、代码块都重新检查过颜色对比度。尤其是 Markdown 代码块，之前在暗色模式下背景和文字对比太低，现在已经加了更明确的文字颜色和背景覆盖。

这次也处理了行内代码和代码块在不同主题里的显示一致性，让它们更容易读，不再只是“看起来有代码块”，而是真的能看清内容。

## 浏览器兼容

Firefox 上曾经出现过顶栏导航显示不全的问题。这个问题和浏览器对 flex 布局、内容宽度计算、缓存策略的处理差异有关。后来给导航设置了更稳定的宽度规则，并给 CSS 和 JS 加了版本参数，减少旧缓存造成的错位。

Chrome、Firefox 和 VS Code 内置浏览器之间的差异，也提醒我：一个静态博客看起来简单，但细节并不总是简单。不同浏览器对同一段 CSS 的理解可能略有不同，尤其是 flex、`max-content`、`:has()` 这类特性。

## 现在的状态

现在这个博客更适合阅读了。顶部导航更清楚，目录能帮助定位，暗色模式更舒服，移动端也不会因为菜单展开而打断阅读。

这些改动没有改变博客的核心：它仍然是一个记录想法的地方。只是现在，写下来的内容可以被更轻松地找到、阅读和回看。
